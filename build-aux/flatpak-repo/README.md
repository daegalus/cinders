# Flatpak Repository Publishing

Cinders is distributed directly through Flatpak. Homebrew is intentionally not part of this flow.

## Local Repo Build

Build an unsigned local repository:

```sh
bash build-aux/flatpak-repo/build-repo.sh
```

If upstream source hosts are temporarily unavailable and the sources are already cached locally, skip network downloads:

```sh
DISABLE_DOWNLOAD=1 bash build-aux/flatpak-repo/build-repo.sh
```

This also skips refreshing the Flathub remote unless `SETUP_FLATHUB=1` is set explicitly.

Install from the local repository:

```sh
flatpak remote-add --user --if-not-exists --no-gpg-verify cinders-local _build/flatpak/repo
flatpak install --user cinders-local dev.yulian.Cinders//stable
```

Rebuild, then update from the same local repository:

```sh
bash build-aux/flatpak-repo/build-repo.sh
flatpak update dev.yulian.Cinders
```

Remove the local test remote:

```sh
flatpak remote-delete --user cinders-local
```

## Published Repo

The published repository is generated by `.github/workflows/flatpak-repo.yml` and deployed to GitHub Pages.

Default URL layout:

```text
https://<owner>.github.io/<repo>/flatpak/
```

Set the repository variable `CINDERS_FLATPAK_BASE_URL` to override that default, for example:

```text
https://daegalus.github.io/cinders/flatpak
```

When `https://yulian.dev/cinders/flatpak` is routed to the Pages deployment,
that can become the value instead.

Users install from:

```sh
flatpak install --user https://daegalus.github.io/cinders/flatpak/dev.yulian.Cinders.flatpakref
```

After install, updates come from:

```sh
flatpak update dev.yulian.Cinders
```

## Signing Key

The public repository workflow requires a dedicated Flatpak signing key. A release repository should not be published unsigned.

Create a dedicated key:

```sh
gpg --quick-generate-key "Cinders Flatpak <flatpak@yulian.dev>" ed25519 sign never
gpg --list-secret-keys --keyid-format=long "Cinders Flatpak"
```

Export the private key:

```sh
gpg --armor --export-secret-key <KEY_ID> > cinders-flatpak-private.asc
```

Set GitHub secrets:

```text
FLATPAK_GPG_PRIVATE_KEY
FLATPAK_GPG_KEY_ID
```

`FLATPAK_GPG_PRIVATE_KEY` should contain the armored private key. `FLATPAK_GPG_KEY_ID` should contain the key id shown by `gpg --list-secret-keys --keyid-format=long`.

Keep this key dedicated to signing the Cinders Flatpak repository.
